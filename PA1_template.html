Subsubsection 4.2.2 A Strategy for Imputing Missing Values

Clearly we can't impute a missing value by looking at it's neigbour,since that neighbor will almost always also be missing.

Now we saw earlier that the steps values varied dramatically across time of day, with a spike at 08:35. So obviously a using a constant value, such as a mean, to imputing the all the values for entire day would be a poor choice. A better choice would be to include the time of day as a factor in our imputation. Since we already have computed the daily average activity, it is easy to use that as our approach, simply replace each missing value with the number of steps predicted by the average activity for that interval.

Subsection 4.3 Creating the The Imputed Data Set

First we create a helper function that returns the average step value given the for a given interval

fn <- function(interval) {
    j <- which(df.ave.activity$interval == interval)  #find the index on activity
    df.ave.activity$average[j]  #return the ave step for that activity
}

Next we construct our new steps set

i.steps <- sapply(1:nrow(df), function(i) {
    ifelse(is.na(df$steps[i]), fn(df$interval[i]), df$steps[i])
})

And finally we form a new data frame with the imputed steps

df.imputed <- df  #copy
df.imputed$steps <- i.steps

Section 5 Differences in activity patterns between weekdays and weekends

Adding a New Factor to our DataSet

To explore the differences in activity patterns between weekday and weekends we add an new factor as an indicator for whether the observation was a weekend or weekday. But first a helper function.

week.day.end <- function(x) {
    y <- weekdays(as.Date(x))
    ifelse(y %in% c("Saturday", "Sunday"), "weekday", "weekend")
}

Next we add a new column to our data set

df$wked <- sapply(df$date, week.day.end)
head(df$wked)

## [1] "weekend" "weekend" "weekend" "weekend" "weekend" "weekend"
